# Example Assessments

This directory contains **fictional** example threat model assessments demonstrating the three assessment types.

## Purpose

These examples illustrate the threat modeling methodology using completely fictitious scenarios, data, and systems. They are designed to:

- Demonstrate the assessment structure and format
- Show how different assessment types (Type 1, Type 2, Type 3) are conducted
- Provide templates for understanding expected deliverables
- Illustrate threat identification and risk scoring approaches

## Assessment Types

### Type 1: Third-Party Vendor Assessment
**Location:** `fictional-vendor-assessment/`

Example assessment of a fictional SaaS vendor (ExampleCorp). Demonstrates:
- Vendor evaluation methodology
- Risk-based vendor recommendation (Proceed / Proceed with Conditions / Do Not Proceed)
- Integration risk analysis
- Third-party dependency assessment

### Type 2: Internal Application Assessment  
**Location:** `fictional-application-assessment/`

Example assessment of a fictional data pipeline. Demonstrates:
- Multi-vendor SaaS chain analysis
- Data flow security evaluation
- Internal system architecture review
- Security requirements derivation

### Type 3: Infrastructure-Only Assessment
**Location:** `fictional-infrastructure-assessment/`

Example assessment of fictional cloud infrastructure. Demonstrates:
- Cloud security posture review
- IAM and network security analysis
- Infrastructure-as-Code security
- Configuration and compliance assessment

## Fictional Nature

All content in this directory is **entirely fictional**:

- Company names (ExampleCorp, ExampleAnalytics, ExampleCloud) are fictitious
- Data types and classifications are illustrative examples
- System architectures represent common patterns, not real systems
- Risk scenarios are representative, not based on actual vulnerabilities
- All findings and recommendations are generic examples

## Using These Examples

These examples demonstrate the **structure** and **methodology** of threat modeling, but should not be used as:

- Actual security assessments
- Vendor evaluation criteria
- Implementation checklists
- Compliance documentation

For real assessments, follow the methodology described in the main documentation and templates.
